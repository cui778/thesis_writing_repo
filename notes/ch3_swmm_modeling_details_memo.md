## 6. DWF 基准流量计算

```text

```

| 符号 | 含义 |
|---|---|

## 7. 日变化与周变化系数

已找到的原始变化系数如下：

```text
[PATTERNS]
;;Name           Type       Multipliers
;;-------------- ---------- -----------
Hourly_STD       HOURLY     0.655 0.600 0.545 0.545 0.600 0.764
Hourly_STD                  0.982 1.091 1.200 1.255 1.200 1.145
Hourly_STD                  1.091 1.091 1.145 1.309 1.473 1.418
Hourly_STD                  1.255 1.145 1.036 0.927 0.818 0.709
;
Week_STD         DAILY      1.029 1.029 1.029 1.029 1.081 0.926 0.875
```

含义：

1. `Hourly_STD` 为 24 h 小时变化系数，用于表达生活污水排放的日内规律。
2. `Week_STD` 为 7 d 日尺度变化系数，用于表达工作日和非工作日之间的负荷差异。
3. 这些系数最初可通过 `[PATTERNS]` 作用于 DWF，也可被脚本展开为 `[TIMESERIES]` 中的显式小时流量序列。

正文推荐写法：

```text
模型在节点 DWF 基准流量基础上引入 24 h 小时变化系数 Hourly_STD，并设置 7 d 日尺度变化系数 Week_STD，使 DWF 输入能够反映早晚峰、低谷时段以及不同日期之间的常态差异。
```

PPT 可画一张 `Hourly_STD` 折线图，展示 24 h 内低谷、上升、高峰和回落过程。

## 8. DWF 到 INFLOWS/TIMESERIES 的转换

当前模型不是简单把 DWF 写成固定常量，而是将水量和水质背景输入统一组织进 `[INFLOWS]` 和 `[TIMESERIES]`。

核心口径：

```text
水量 FLOW 与污染物浓度 CONCEN 统一由 INFLOWS 调用 TIMESERIES 驱动。
```

这样设计的目的：

1. 避免正常工况成为静态常量输入。
2. 使 FLOW 和 CONCEN 都能随时间变化。
3. 便于叠加后续 I/E 缺陷注入。
4. 使 baseline、缺陷场景和 residual 计算具有统一时间坐标。
5. 便于后续扩展新的污染物、示踪变量或缺陷类型。

当前正式 INP 中可以看到：

```text
[INFLOWS]
节点  FLOW  DWF_节点名  FLOW
节点  BODf  BGConc_DEFAULT_BODf  CONCEN
节点  NH4   BGConc_DEFAULT_NH4   CONCEN
...
```

当前正式 INP 中 `[TIMESERIES]` 包含一周长度的显式小时序列。正式仿真时段为 48 h，因此只调用该一周序列中的前 48 h 片段。

## 10. 水质模拟总体思路

用户找到的旧版本说法是：

```text
水量-浓度同通道 + 节点机理反应 + 管道保守传输
```

这个说法的方向是对的，但不完全符合当前正式 INP。根据当前 `2_tuned_v3_merged1.inp`，更准确的口径应为：

```text
水量-浓度同通道输入 + 管网输移 + Storage 节点反应为主 + 部分组分管道一阶衰减
```

原因是当前正式模型中：

1. BODf、BODs、TSSs 在 `[POLLUTANTS]` 中设置了 Kdecay，不是完全保守传输。
2. NH4、NO3、DO、TSSn、TR_in、TR_ww 在 `[POLLUTANTS]` 中 Kdecay = 0，管段中可近似视为随水流输移。
3. NH4/NO3/DO 的主要转化通过 `[TREATMENT]` 在 Storage 节点中表达。
4. 当前正式模型为 13 个 Storage 节点，不是旧版本记录中的 18 个。

## 11. 当前正式 INP 的污染物设置

当前 `[POLLUTANTS]` 中共有 9 种组分：

| 组分 | 单位 | Kdecay | 初始/背景参考值 | 当前作用 |
|---|---|---:|---:|---|
| BODf | MG/L | 0.30 | 200 | 快速可降解有机物衰减 |
| BODs | MG/L | 0.05 | 50 | 缓慢可降解有机物或水解过程 |
| NH4 | MG/L | 0.00 | 30 | 管道中不设一阶衰减 |
| NO3 | MG/L | 0.00 | 0 | 管道中不设一阶衰减 |
| DO | MG/L | 0.00 | 2 | 管道中不设一阶衰减 |
| TSSs | MG/L | 0.10 | 150 | 可沉降悬浮物衰减 |
| TSSn | MG/L | 0.00 | 100 | 不沉降悬浮物或背景悬浮组分 |
| TR_in | MG/L | 0.00 | 0 | 示踪变量 |
| TR_ww | MG/L | 0.00 | 1 | 示踪变量 |

重要差异：

当前正式模型没有在 `[POLLUTANTS]` 中设置：

```text
NH4 Kdecay = 0.15
CoPollutant = NO3
CoFrac = 1.0
```

也就是说，当前正式模型没有通过 SWMM 的 CoPollutant 机制在管道中直接设置 NH4 衰减生成 NO3。

## 12. 当前正式 INP 的 TREATMENT 设置

当前 `[TREATMENT]` 对 13 个 Storage 节点设置处理反应，共 117 条公式，即：

```text
13 个 Storage 节点 x 9 个污染物 = 117 条
```

主要公式如下：

| 组分 | 当前公式 | 说明 |
|---|---|---|
| BODs | `C = BODs*exp(-0.15*HRT)` | Storage 节点内有机物衰减 |
| BODf | `C = BODf - 2.0*BODf*HRT` | Storage 节点内快速有机物消耗 |
| NH4 | `C = NH4 - 20.0*NH4*HRT` | Storage 节点内氨氮削减 |
| NO3 | `C = NO3 + 20.0*NH4*HRT - 5.0*NO3*HRT` | Storage 节点内硝酸盐生成与削减 |
| DO | `C = DO + 30.0*(9.1-DO)*HRT - 2.0*BODf*HRT` | Storage 节点内复氧与耗氧 |
| TSSs | `C = TSSs*exp(-0.3*HRT)` | Storage 节点内悬浮物沉降 |
| TSSn | `C = TSSn` | 保持不变 |
| TR_in | `C = TR_in` | 示踪变量保持不变 |
| TR_ww | `C = TR_ww` | 示踪变量保持不变 |

正文中可以写“有机物降解、氮素转化、复氧和悬浮物去除等过程”，但不要过度声称精细水质机理率定已经完成。

## 14. I/E 缺陷模拟与水质关系

当前 IE420 缺陷矩阵中，I/E 缺陷本质上是水力扰动：

1. I 类渗入：正向入流扰动。
2. E 类渗漏：负向流量扰动。
3. PySWMM 中通过 `generated_inflow(flow)` 动态注入。
4. 根据 `start_hour` 和 `duration_h` 控制缺陷活跃时段。
5. 当前 I/E 不额外注入独立污染源。

因此正文应写：

```text
本文中的 I/E 缺陷以水力扰动方式构造，不额外设置独立污染源输入；水质变量作为 SWMM 水质输移过程中的伴随输出，与水力变量共同构成节点时序响应。
```

不要写成“缺陷直接引起污染物源项变化”。更稳的说法是：

```text
水量扰动会改变节点入流、出流和停留时间条件，从而可能使 NH4、TSSs 等水质变量表现出伴随响应。
```

该判断已有探索性验证图支持，但正文不宜过度展开机理解释。

这说明正式母数据保留全网节点，而不是只导出监测点。

## 21. 输出变量与 residual 特征

水力输出字段包括：

```text
depth
head
volume
lateral_inflow
total_inflow
total_outflow
flooding
```

水质输出包括各污染物浓度，后续正式诊断实验主要关注：

```text
NH4
TSSs
```

Residual 定义：

```text
residual[t, n, f] = defect[t, n, f] - normal[t, n, f]
```

主要特征：

```text
depth_residual
total_outflow_residual
pollut_NH4_residual
pollut_TSSs_residual
*_residual_rel
```

正文中保留“原始时序 + residual 派生特征”这一稳妥口径，不把 residual 写成唯一输入。

## 22. 单 baseline 问题的备用解释

正文不要主动强调“只有一个 baseline”。如果有人质疑现实中正常工况也会随机波动，可使用以下备用解释：

```text
本文中的 baseline 并非静态常数工况，而是在给定 DWF 日变化、背景水质浓度和边界条件下形成的动态正常工况。Residual 特征的作用不是假定现实只存在一种正常状态，而是在受控仿真实验中提供同一模型、同一边界和同一时间输入下的参照响应，用于突出 I/E 缺陷导致的增量变化。
```

如果后续需要增强说服力，可做附加实验：

1. 多 DWF 扰动 baseline。
2. 原始时序 + residual 辅助输入。
3. 不同噪声强度下的鲁棒性检查。

但当前第3章正文暂不主动展开这个问题。

## 23. 和张超论文相比的差异

张超论文可借鉴的部分：

1. 先写研究对象和数据来源。
2. 再写 SWMM 数值模拟方法。
3. 再写 PySWMM 自动化仿真。
4. 最后写多场景数据生成。

不应照搬的部分：

1. DEM、土地利用、土壤参数等流域产汇流参数。
2. 降雨径流过程的细致水文参数。
3. 长时间月/周变化模拟口径。
4. 与本文 I/E 缺陷诊断无关的模型性能章节。

本文可以补充但不必过度展开的内容：

1. 研究子区划分图。
2. 管网拓扑图。
3. DWF 日变化图。
4. SWMM 稳定性报告表。
5. 典型 I/E 场景响应图。
6. 水质输入和反应设置表。

## 24. PPT 中可展示的内容

中期答辩 PPT 的第3部分建议展示：

1. 研究子区与资料来源：放区域图或管网示意图。
2. SWMM baseline 构建流程：工程资料整理、子区划分、结构概化、DWF 注入、边界设置、稳定性检查。
3. DWF 日变化输入：放 `Hourly_STD` 或代表节点 DWF 曲线。
4. 模型规模与稳定性：128 节点、137 管段、连续性误差表。
5. IE420 缺陷协议：1 baseline + 250 I + 170 E。
6. PySWMM 批量仿真流程：读取矩阵、时间门控、`generated_inflow`、10 min 采样。
7. 数据集规模：421 场景、287 时间点、36736 条/场景。
8. 典型缺陷响应：水力 residual 曲线，可附 NH4/TSSs 伴随响应图。

PPT 不建议放太多水质反应公式，最多放“水量-浓度同通道 + Storage 节点反应”的概念图或简表。
