# 单 baseline 质疑的补充实验计划

## 1. 实验目标

当前主实验采用如下受控仿真差分思路：

```text
缺陷场景响应 - 无缺陷正常场景响应 = residual 特征
```

该设计的优点是能够突出 I/E 缺陷引起的增量响应，削弱节点固有水力差异。但潜在质疑是：现实中的正常工况本身存在 DWF 日变化、用水随机波动、水质浓度波动和边界条件变化，仅使用一个正常场景作为参照，可能使 residual 对正常波动过于敏感。

因此，补充实验的目标是：

> 检验模型性能是否依赖单一正常参照工况，并评估在多个正常背景条件下，residual 特征和诊断模型的稳定性。

## 2. 核心思路

不直接废除现有 IE420 主实验，而是在现有 IE420 协议外，构造一组规模较小的多 baseline 鲁棒性实验。

核心设计为：

```text
多个正常背景工况
×
一组代表性 I/E 缺陷场景
→
分别构造 residual
→
比较诊断性能和残差分布稳定性
```

也就是说，主实验仍然使用 IE420；补充实验只回答一个问题：

> 如果正常工况换了，缺陷响应是否仍然可识别？

## 3. 多 baseline 设计

可以构造 3 到 5 个正常背景工况，不需要一开始就做很多。

| baseline 类型 | 设计方式 | 目的 |
|---|---|---|
| B0 标准正常工况 | 当前 DWF 日变化曲线 | 作为现有主实验参照 |
| B1 低负荷正常工况 | DWF 整体降低 10%-20% | 模拟低用水日 |
| B2 高负荷正常工况 | DWF 整体提高 10%-20% | 模拟高用水日 |
| B3 峰值偏移工况 | 早晚峰时间提前或延后 1-2 h | 模拟用水节律变化 |
| B4 噪声增强工况 | 在 DWF 与水质浓度上叠加小幅随机扰动 | 模拟正常随机波动 |

初期可以只做：

```text
B0 + B1 + B2
```

如果结果稳定，再扩展到 B3、B4。

## 4. 缺陷场景抽样

不建议一开始对 IE420 全量重跑多 baseline，因为计算量会明显增加。

建议先抽取一个代表性子集，例如 IE60 或 IE100。

### IE60 方案

| 类型 | 数量 | 说明 |
|---|---:|---|
| I 类渗入 | 30 | 覆盖不同拓扑位置和强度 |
| E 类渗漏 | 30 | 从合法 E 节点中抽取 |
| 合计 | 60 | 用于鲁棒性检验 |

### IE100 方案

| 类型 | 数量 | 说明 |
|---|---:|---|
| I 类渗入 | 60 | 覆盖更多候选节点 |
| E 类渗漏 | 40 | 保持与正式协议相近比例 |
| 合计 | 100 | 结果更稳，但计算量更大 |

建议从 IE60 开始。

## 5. 实验组合

如果采用 3 个 baseline 和 IE60，则总仿真量为：

```text
3 个 baseline + 3 × 60 个缺陷场景 = 183 个场景
```

如果采用 5 个 baseline 和 IE100，则总仿真量为：

```text
5 个 baseline + 5 × 100 个缺陷场景 = 505 个场景
```

所以第一阶段更推荐：

```text
B0/B1/B2 × IE60
```

这是一个比较轻的验证实验。

## 6. Residual 构造方式

每个缺陷场景必须和对应背景 baseline 配对，而不是都减同一个 B0。

例如：

```text
residual_B1 = defect_under_B1 - baseline_B1
residual_B2 = defect_under_B2 - baseline_B2
```

也就是说，背景工况变了，缺陷工况和正常工况要一起变，形成配对对照。

不建议采用：

```text
defect_under_B2 - baseline_B0
```

这种方式会把正常背景差异混进缺陷 residual 中，导致残差不再只表示缺陷引起的增量响应。

## 7. 对照实验设计

可以设计三组对照。

### 方案 A：单 baseline 训练与测试

```text
训练：B0 residual
测试：B0 residual
```

作用：复现当前主实验逻辑。

### 方案 B：跨 baseline 测试

```text
训练：B0 residual
测试：B1/B2 residual
```

作用：检验模型是否过拟合 B0 背景。

如果性能明显下降，说明单 baseline 敏感。

### 方案 C：多 baseline 混合训练

```text
训练：B0 + B1 + B2 residual
测试：留出某一个 baseline
```

例如：

```text
训练：B0 + B1
测试：B2
```

作用：检验多背景训练能否提升对正常波动的鲁棒性。

## 8. 评价指标

不需要重新设计所有指标，可以沿用第4章已有指标。

| 指标 | 用途 |
|---|---|
| MRR | 缺陷节点排序整体质量 |
| Top-1 | 第一预测是否命中 |
| Top-3 | 工程排查范围是否可控 |
| Top-5 | 宽松定位能力 |
| active period recall | 缺陷活跃期识别能力 |
| event-level Top-K | 场景级定位能力 |

另外可以增加两个数据层面的指标。

| 指标 | 用途 |
|---|---|
| residual energy 分布差异 | 看不同 baseline 下缺陷响应是否稳定 |
| active/inactive residual ratio | 看缺陷活跃期是否仍明显区别于非活跃期 |

## 9. 预期结果解释

### 情况 1：跨 baseline 性能变化很小

可以说明：

> 当前 residual 特征主要捕捉 I/E 缺陷引起的增量扰动，对正常 DWF 背景的小幅变化不敏感，单正常工况参照在本文受控仿真设置下具有可接受的稳定性。

### 情况 2：跨 baseline 性能下降，但多 baseline 训练恢复

可以说明：

> 模型对正常背景变化存在一定敏感性，但通过引入多正常背景工况进行训练，可提升诊断模型对 DWF 波动的鲁棒性。

### 情况 3：跨 baseline 和多 baseline 都下降明显

可以说明：

> 当前 residual 设计对背景工况依赖较强，后续需要转向原始时序 + residual、多工况训练，或显式加入 DWF 背景参数作为模型输入。

## 10. 推荐执行路线

建议后续按三步推进：

```text
第一步：保留当前 IE420 主实验
第二步：补充 B0/B1/B2 × IE60 鲁棒性实验
第三步：如果结果有必要，再扩展到 B0-B4 × IE100
```

这样既不需要立即推翻现有实验，也可以为“单 baseline 是否可靠”这一质疑准备可执行的补充验证。当前论文正文可先低调写成“原始时序 + residual 派生特征”；等补充实验完成后，再决定是否在第3章或第4章增加“多正常背景鲁棒性检验”小节。
